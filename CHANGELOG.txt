## [1.1.0] — 2026-07-16
Agent: Claude
Intent: Modern-hardware feature update — detect + adapt to X3D CPUs, Resizable
        BAR, DDR5, NVMe/SSD storage; smarter warm cache; DllMain safety fix.
Files changed:
  - source/src/main.cpp (version 1.1.0; all features below)
  - source/CMakeLists.txt (VERSION 1.1.0; +setupapi.lib +cfgmgr32.lib)
  - source/package/SKSE/Plugins/NextGenDiskCache.ini ([Hardware] section;
    iWarmCacheBudgetMB / iWarmCacheThreads / bWarmCacheMappedPrefetch /
    bWarmCacheLowIoPriority)
  - source/README.md ("1.1.0 — modern hardware support" table)
  - NextGen Disk Cache 1.1.0/SKSE/Plugins/NextGenDiskCache.dll (348160 bytes)
  - NextGen Disk Cache 1.1.0/SKSE/Plugins/NextGenDiskCache.ini + .pdb + README.md
New features:
  - Hardware profiler (logged at startup): CPU brand/cores/L3 topology with
    X3D (3D V-Cache) + dual-CCD + Intel hybrid P/E detection; RAM DDR
    generation + configured MT/s via SMBIOS Type 17; game-drive class
    NVMe/SATA SSD/HDD via seek-penalty + bus-type IOCTL; GPU largest PCI BAR
    via SetupAPI/cfgmgr32 (ResType_Mem + ResType_MemLarge) → Resizable BAR
    ACTIVE/OFF verdict; real OS build via RtlGetVersion.
  - bAutoTune: warm-cache budget by RAM tier (8G/4G/1.5G/512M, ¼-of-free cap,
    256 MB HDD cap), parallel readers by drive (NVMe 4 / SATA 2 / HDD 1),
    per-file cap by drive (64/32/8 MB).
  - Warm cache rework: multi-threaded work-stealing over size-sorted archive
    list; MapViewOfFile + PrefetchVirtualMemory (ReadFile-loop fallback);
    IoPriorityHintLow on warm handles; EcoQoS allowed on warm threads.
  - bPreferFastCores (default OFF): SetProcessDefaultCpuSets soft hint to
    V-Cache CCD (dual-CCD X3D) or P-cores (Intel hybrid).
  - Fixes: MessageBoxA removed from DllMain (loader-lock deadlock risk —
    now logged instead); ClassifyPathW no longer truncates long paths (wide
    extension extraction); warm budget RAM tiers carry slack for reserved
    memory (31.8 GiB "32 GB" kits); log strings pure ASCII (no mojibake);
    new kernel32 APIs resolved via GetProcAddress so Win7/8.1 still loads.
Why / root cause:
  - User asked for modern-PC tech support (x3D, ReBAR, DDR5-6000 CL30, SSD DRAM).
    BIOS-level tech cannot be enabled from software, but it CAN be detected,
    reported, and used to auto-tune the software-side cache behavior.
Verify:
  - build.ps1 (MSVC 19.51, VS 18 2026, Release x64) → BUILD OK, 0 errors;
    only pre-existing vendored warnings (IPrefix.h C4005, detours LNK4099)
  - dumpbin /exports: SKSEPlugin_Version + SKSEPlugin_Query + SKSEPlugin_Load
  - Runtime harness (LoadLibrary + SKSEPlugin_Load + fake Data\ archives) on
    the real rig: detected 9800X3D 96MB L3 [3D V-Cache], DDR5-6000 2 modules,
    NVMe SSD, RTX 4080 SUPER BAR 16384 MB → ReBAR ACTIVE, Win11 26200;
    auto-tune plan budget 3604 MB / 3 threads / mapped prefetch; warmed
    32 MB across 3 files; clean exit stats. Hooks attach/detach clean.
  - UNPROVEN: in-game run under real SKSE (user will test).
  - Zip: NextGenDiskCache-1.1.0.zip (SHA256 D25FCFB8…66C6A9BA)
Nexus: NO
