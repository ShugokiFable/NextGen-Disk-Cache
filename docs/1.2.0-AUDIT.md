# NextGen Disk Cache 1.2.0 source audit

## Release status

This is a source release candidate. Static transformation, configuration, XML, packaging, and policy checks pass in the supplied validator. A Windows MSVC build and Skyrim runtime test remain required before a Nexus binary is called release-ready.

## Correctness fixes

- Moved settings I/O, log creation, dynamic API resolution, and Detours attachment out of `DllMain`.
- Replaced racy load/subtract warm-budget accounting with atomic reservation and refund.
- Capped fallback `ReadFile` requests to the remaining per-file budget.
- Restricted forced `FILE_FLAG_RANDOM_ACCESS` to recognized asset extensions rather than unknown files.
- Disabled working-set expansion by default because Skyrim's process working set is not the system file cache.
- Replaced exit-time file logging with a safe post-warm statistics snapshot.
- Moved release PDBs outside the public package and retained them as a separate CI artifact.

## High-end behavior

- High-end auto-tune is the default FOMOD profile.
- NVMe systems receive larger distributed archive windows, up to eight low-priority warm workers, and adaptive budgets with a free-RAM reserve.
- Dual-CCD X3D systems place background warm workers on the smaller/non-V-Cache CCD when Windows exposes matching CPU sets. Single-CCD X3D systems are left unconstrained.
- Hybrid CPUs place warm workers on the lowest efficiency class when available.
- DDR generation and configured MT/s are logged through SMBIOS. CAS latency is not invented.
- ReBAR is diagnostic only for this CPU/file-cache plugin.
- SSD DRAM/SLC behavior remains firmware-managed and is not presented as a controllable feature.

## DirectStorage

- DirectStorage 1.3 is dynamically loaded, so the plugin has no hard runtime dependency.
- Probe-only mode validates the factory without creating an idle queue.
- The experimental backend creates a low-priority file-to-system-memory queue and performs real batched reads.
- Cancellation waits for the status entry before scratch memory and file objects are released.
- Raw DirectStorage remains off by default because it is not equivalent to warming the normal Windows file cache.

## Unproven gates

- MSVC compilation against the NuGet 1.3 header.
- SKSE load on supported Steam and GOG runtimes.
- Startup cancellation and rapid process exit.
- Dual-CCD X3D CPU-set behavior on physical hardware.
- DirectStorage absent, bundled, and provided by another component.
- Performance benefit versus the page-cache backend on representative mod lists.
