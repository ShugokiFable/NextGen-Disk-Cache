# NextGen Disk Cache 1.2.0 technical notes

## What hardware support means

The plugin cannot enable 3D V-Cache, Resizable BAR, XMP/EXPO, DDR5 CAS latency, or an SSD's onboard DRAM. Those are CPU, BIOS, memory-controller, GPU-driver, or SSD-firmware properties. Version 1.2 uses observable topology to avoid wasting those resources:

- Dual-CCD X3D: background warm threads prefer the smaller/non-V-Cache CCD, leaving the V-Cache CCD for foreground game work.
- Single-CCD X3D: no affinity policy is applied because every core shares the large cache domain.
- Intel hybrid: warm threads prefer the lowest efficiency class when CPU-set data is available.
- DDR5-6000: detected and logged through SMBIOS configured speed. CL30 is SPD data and is not fabricated.
- ReBAR/SAM: detected and logged. It does not accelerate CPU-to-RAM file-cache reads by itself.
- NVMe: receives higher concurrency, larger distributed prefetch windows, and an optional DirectStorage path.
- SSD DRAM/SLC: cannot be queried reliably as a generic capability. Sequential/batched reads may naturally exercise it, but the mod does not claim direct control.

## DirectStorage behavior

`bDirectStorageProbe=1` dynamically loads `dstorage.dll` and validates an `IDStorageFactory`. A low-priority system-memory queue is created only when `bDirectStorageWarmRead=1`, so capability logging does not spin up an idle DirectStorage queue. The core mod still functions when the runtime is absent.

`bDirectStorageWarmRead=1` is experimental. It submits real file-to-memory DirectStorage requests in batches and discards the scratch memory. Cancellation waits for the queued status entry before releasing scratch buffers. This is useful for raw path testing and can touch controller caches, but it is not treated as a replacement for `PrefetchVirtualMemory`, which is the path designed to populate Windows' normal memory manager/file-cache state.

## Release gate

Do not publish the resulting DLL until the Windows workflow succeeds and these are tested in Skyrim:

1. Startup and shutdown with SKSE on 1.5.97, 1.6.640, and the supported GOG runtime.
2. DirectStorage runtime absent, bundled, and present through another mod.
3. Warm-cache cancellation under low available RAM.
4. Dual-CCD X3D and single-CCD X3D logs.
5. A heavy mod list containing large BSAs, loose files, BSAMemoryMap, and Fast Decompress Skyrim.
6. Confirm no PDB, LIB, EXP, or private build paths enter the Nexus ZIP.
